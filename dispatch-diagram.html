03 — Dispatch Lifecycle
From "Send" click to task completion
1
Sender composes
Picks a friend, types task, hits Send → POST /dispatch to broker
2
Broker routes
Stores in Postgres, pushes new_dispatch over WS to receiver's daemon
3
Receiver notified
macOS notification fires, consent window opens at localhost:8001
4
Accept / Reject
Receiver clicks Accept → daemon starts Claude agent. Reject → sender sees "denied"
5
Agent runs
Claude executes task. Read-only tools run freely. Write/Edit/Bash → pause
6
Tool consent
Receiver sees tool + JSON input → Allow / Deny (120s timeout → auto-deny)
7
Events stream
All events flow: broker → sender's dispatch card in real time
8
Done / Error
Final status: completed, failed, denied, or expired. Sender sees cost + turns.

05 — WebSocket Channels
WS /agent/connect
Daemon → Broker
Daemon connects here on startup. Broker pushes new_dispatch when a task arrives. Daemon streams dispatch_event and dispatch_status back.
WS /dispatch/:id/watch
Sender browser → Broker
Sender watches one dispatch. Broker replays past events on connect then streams new ones in real time.
WS /inbox
Broker → Recipient browser
Unified web inbox. Receives inbox_new on new dispatch plus all per-dispatch events. Sends dispatch_decision and tool_approval back.
WS /ws/inbox (local)
Daemon → Consent UI
Local-only. Snapshot of all pending dispatches on connect. Receives dispatch_decision and permission_response from the local consent UI.
WS /ws/dispatch/:id (local)
Daemon → Consent UI
Watches one dispatch locally. Replays events, streams new ones. Receives permission_response for tool approval.
06 — Event Types
agent_text
Daemon → All
Streaming text from Claude as it thinks/responds
tool_use
Daemon → All
Tool name + full JSON input the agent is calling
tool_result
Daemon → All
Output from tool execution, with truncated flag
permission_request
Daemon → Recipient
Destructive tool needs approval — id, tool, input
permission_response
Recipient → Daemon
allow or deny decision for a specific request_id
dispatch_status
Daemon → All
pending → delivered → accepted → running → completed / failed / denied / expired
done
Daemon → All
Final: subtype, duration_ms, num_turns, total_cost_usd
error
Daemon → All
exception class name + message string
07 — Missing / Needs Building
🚨
Consent UI Frontend (Critical)
web/recipient/ was deleted in the merge. The tray app opens localhost:8001 for incoming tasks but there is no HTML/JS to display. The local_app.py API exists — just needs a frontend.
🔄
WebSocket Auto-Reconnect
Sender UI drops events if the WS disconnects mid-task. No reconnect logic exists in app.js.
⚙️
Settings / Account Page
No UI to change API key, workspace path, or broker URL after onboarding.
🔔
Tray Notification Badge
No badge count on the menu bar icon when dispatches are pending approval.
📋
Dispatch History (Sender UI)
The API GET /dispatches?role=sent exists but the sender UI has no history tab.
⏱
Expiry Countdown
Receiver has 5 min to accept/reject but the consent UI shows no countdown timer.
🌐
Remote Broker URL
BROKER_URL is hardcoded to localhost:8000. Must point to Railway for two users to communicate.
👥
Unfriend / Remove Friend
No endpoint or UI to remove a friend once added.
